The Description of the security issues show that the total security issues are thus

Remote Code Execution: This is usually a critical vulnerability in most systems and it allows an attacker to remotely executes malicious code on a computer and have control of the computer

Arbitrary Code Execution: This is similar to remote code execution but the difference between them is that the latter involves running the mailcious code or command from over the internet.

Deserialization of Untrusted Data: Data input that are not properly formatted will generate untrusted possibility. This possibility could be used to abuse application logic, execute arbitrary code and deny service.

Directory Transveral: This is a web vulnerability that allows an attacker to be able to read files that are not suppose to out-of-bounds. These files could be for back-end system or sensitive files and they could lead to the attacker taking over the system by modifying this system

Denial of Service (DoS): Just as it names implies, this vulnerability enables the attacker to shutdown a system in order to make the services accessible to the intended users

Server Side Template Injection (SSTI): This vulnerability gives an attacker the leverage to inject malicious payload into a native template syntax. This is possible when the user input is concatenated into the template directly.

Access Control Bypass: This is bypassing access control checks by modifying internal application state, modifying the website URL or the internal application state

Unrestricted Upload of File with Dangerous Type: When this vulnerability is exploited it allows an attacker to transfer or upload files that can automatically processed within the production environment

XML External Entity (XXE): This vulnerability would allow and attacker to interfere with the XML data of an application that is processing it. This could lead to attacker to be able to view files on the application server and interact with back-end system it can access

SQL Injections: This is a database vulnerability which is common amongst all most website and it allows an attacker ti interfere with the queries that the application would be making with the database

Expression Language Injection: When an attacker can control the data that enters the EL interpreter, we could say that this vulnerability exists.

Improper Authorization: These bugs occurs when there is no checks in the access functionality of a use in an application. By doing this an attacker can modify or delete file that they are not supose to.

Privilege Escalation: This is a misconfiguration that can be exploited to have elevated rights in access to resources that would be normally protected from an application user

Cross-site Request Forgery (CSRF): If this vulnerability is present, it would enable an attacker to force an end user to execute unwanted actions without their awareness. They are sort of us exectued with help from using social engineering

Cross-site Scripting (XSS): In this vulnerability an attacker will be able to send malicious code to different end users using a form of browser side scripts and these scripts can be used to access any cookies, session tokens or any sensitive informaation that would be retained by the web application
